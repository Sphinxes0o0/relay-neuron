# 研究内容 Review 系统

## Review 检查清单

### 1. 内容深度检查
- [ ] 每个章节至少有 500 字以上的论述
- [ ] 每个研究发现都有具体数据（样本量、p值、效应量等）
- [ ] 包含研究的局限性讨论
- [ ] 有对比研究或反驳观点

### 2. 引用规范检查
- [ ] 每项结论标注 DOI 或稳定 URL
- [ ] 格式: `([期刊/作者, 年份](链接))`
- [ ] 引用数量: 每份报告至少 5 篇以上参考文献

### 3. 结构完整性检查
- [ ] 有研究摘要/概述
- [ ] 有背景/引言
- [ ] 有方法/研究发现
- [ ] 有讨论/结论
- [ ] 有参考文献列表

### 4. 可操作性检查
- [ ] 临床建议具体可执行
- [ ] 证据等级标注清晰
- [ ] 适用范围明确

---

## Review 任务模板

```markdown
## [文件名] Review 报告

### 问题列表
| # | 问题 | 严重程度 | 建议修复 |
|---|------|----------|----------|
| 1 | 章节X内容过少 | 高 | 补充... |
| 2 | 缺少具体数据 | 中 | 添加p值、样本量等 |

### 总体评分
- 内容深度: ★★★☆☆
- 引用规范: ★★★★★
- 结构完整: ★★★★☆
- 可操作性: ★★★★☆

### 修复优先级
1. [最高优先级问题]
2. [次要问题]
```

---

## Agent Review Prompt

当用户发送 "review topic: [文件名]" 时：

1. 读取目标 md 文件
2. 对照检查清单逐项审核
3. 输出 Review 报告
4. 如内容不充分，补充完善
5. 保存修复后的文件

---

## 自定义 Review 任务队列

当用户发送 "review all" 时，执行全量 review：

1. 扫描 `research/` 目录下所有 md 文件
2. 按检查清单逐项审核
3. 输出每个文件的 Review 报告
4. 对不充分内容进行补充
5. 完成后 git commit
